# Кейс 1: Создание PRD

## Ситуация
Ты — продакт-менеджер приложения LoyalBox (программа лояльности для ритейл-сети). Нужно написать PRD на новую фичу «Сторис» для обсуждения с командой. У тебя есть бриф от бизнеса, контекст приложения и шаблон PRD по которому нужно оформить документ.

## Данные
- `data/app_context.md` — описание продукта, метрики, команда
- `data/feature_brief_stories.md` — бриф на фичу от бизнеса
- `data/prd_template.md` — шаблон PRD (структура документа)

## Промпт для Cursor

```
Прочитай файлы:
- data/app_context.md — контекст нашего приложения LoyalBox
- data/feature_brief_stories.md — бриф на фичу «Сторис»
- data/prd_template.md — шаблон PRD

Напиши PRD на фичу «Сторис» в файл prd_stories.md, строго по структуре шаблона:

1. Обзор продукта — описание фичи и роль в экосистеме LoyalBox
2. Ценностные предложения — для пользователей и для бизнеса
3. Целевые метрики — таблица KPI с конкретными целевыми значениями из брифа
4. Функциональные требования — навигация, типы сторис, метаданные, бизнес-правила (макс 7 одновременно и т.д.)
5. Администратор — CMS для создания сторис, статусы, аналитика
6. Технические зависимости — что нужно от бэкенда, CDN, аналитики
7. Сценарии использования — основной + альтернативные (пользователь, коммерческий менеджер)
8. Экраны и макеты — список требуемых экранов для дизайнера
9. Фазы разработки — MVP (3 недели) и Post-MVP
10. Открытые вопросы — таблица со статусами

Заполни все секции конкретными данными из брифа и контекста. Пиши структурно, без воды.
```

## Что демонстрирует
- Cursor работает с тремя файлами: бриф + контекст + шаблон
- Генерирует PRD по заданной структуре, не выдумывая свою
- Заполняет шаблон конкретными данными из исходников
- Экономит 2-3 часа на написание PRD

## Время: 2-3 минуты
