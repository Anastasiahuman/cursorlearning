# Кейс 3: Глубокий анализ retention — «Какие фичи удерживают пользователей?»

## Контекст для участников
CPO ставит задачу: **D7 retention = 21.5% — это ниже бенчмарка для category (25-30%). Найдите, какие фичи больше всего влияют на retention, и предложите план роста до 28% за полгода.**

## Что делаем в Cursor (пошагово)

### Шаг 1. Смотрим текущее состояние retention
```
Проанализируй cohorts/cohort_retention.json.
Покажи тренд D1, D7 retention по когортам.
Retention растёт или падает? Какой темп улучшения?
```

### Шаг 2. AI показывает тренд
Ожидаемый ответ:
- D1 retention растёт: 40.2% (W45) → 45.5% (W04), +0.53 pp/неделю
- D7 retention растёт: 17.8% (W45) → 22.0% (W03), +0.47 pp/неделю
- Тренд положительный, но для достижения 28% нужно ускорить рост

### Шаг 3. Ищем драйверы retention
```
Посмотри features/feature_metrics.json и
cohorts/cohort_retention.json (retention_by_segment).
Какие фичи и действия пользователя в первый день
сильнее всего коррелируют с D7 retention?
Составь рейтинг от самого сильного к слабому.
```

### Шаг 4. AI строит рейтинг влияния
Ожидаемый ответ:
| Фича/действие | D7 с фичей | D7 без | Lift |
|---|---|---|---|
| Mini-Games (день 0) | 35.8% | 10.2% | +251% |
| Daily Quests | 35.8% | 12.0% | +198% |
| Family Profile | 38.2% | 14.5% | +163% |
| Leaderboard | 48.2% | 18.5% | +160% |
| Completed Onboarding | 32.5% | 12.8% | +154% |

### Шаг 5. Строим стратегию
```
На основе этих данных предложи стратегию роста
D7 retention с 21.5% до 28%. Учти:
- feature_metrics.json (adoption rate каждой фичи)
- experiments/ab_tests.json (что уже тестировали)
- Что можно увеличить: adoption ключевых фич в первые 24ч

Формат: таблица с инициативами, target metric, expected lift.
```

### Шаг 6. Считаем бизнес-эффект
```
Если D7 retention вырастет с 21.5% до 28%,
как это повлияет на MAU, DAU и revenue через 3 месяца?
Используй текущие метрики из metrics_data.json для расчёта.
```

## Чему учит этот кейс
- **Когортный анализ**: как читать когортные таблицы и видеть тренды
- **Feature-level analytics**: связь фич с удержанием — основа приоритизации
- **Корреляция ≠ причинность**: AI может подсветить, что correlation matrix ≠ causation
- **Бизнес-моделирование**: от метрики → к деньгам
