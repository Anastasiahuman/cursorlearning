# Cursor Training: Кейсы для продакт-менеджера

Репозиторий для обучения работе в Cursor. Контекст — мобильное приложение программы лояльности **LoyalBox** для ритейл-сети.

## Кейсы

| # | Кейс | Что делаем | Время |
|---|------|-----------|-------|
| 1 | [Создание PRD](cases/case_1_prd.md) | PRD на фичу «Сторис» из брифа | 2-3 мин |
| 2 | [Презентация по метрикам](cases/case_2_presentation.md) | HTML-презентация с графиками из JSON | 3-4 мин |
| 3 | [Работа с таблицами](cases/case_3_tables.md) | Дашборд по кампаниям: расчёт ROI, сводка | 3-4 мин |
| 4 | [Анализ документации](cases/case_4_docs_analysis.md) | Саммари API + GAP-анализ для новой фичи | 3-4 мин |
| 5 | [Связка Notion + n8n](cases/case_5_notion_n8n.md) | Автоматизация еженедельной отчётности | 4-5 мин |
| * | [Карусель для соцсетей](cases/case_bonus_carousel.md) | Экспертная карусель из статьи | 3-4 мин |

> **Кейс 2 — демонстрационный (WOW).** Из сырого JSON за 3 минуты получается интерактивная презентация с графиками. **Бонус-кейс** — из статьи в стильную карусель. Остальные — практические задачи из реальной работы PM.

## Данные

Все данные в папке `data/`:
- `app_context.md` — описание продукта LoyalBox, метрики, команда
- `metrics_jan2026.json` — метрики за январь 2026
- `campaigns_table.csv` — маркетинговые кампании
- `feature_brief_stories.md` — бриф на фичу «Сторис»
- `prd_template.md` — шаблон PRD (структура документа)
- `api_docs_loyalty.md` — документация API
- `n8n_notion_context.md` — контекст для автоматизации
- `article_loyalty_trends.md` — статья для карусели

## Начало работы

**Первый раз в Cursor?** Начни с [инструкции по настройке](setup_guide.md) — там всё по шагам, простым языком.

## Как использовать

1. Открой репозиторий в Cursor ([как это сделать](setup_guide.md#шаг-4-открой-папку-с-кейсами))
2. Открой описание кейса из папки `cases/`
3. Скопируй промпт в чат Cursor (`Cmd+L` / `Ctrl+L`)
4. Смотри результат в браузере или в файлах

## Лендинг интенсива «Cursor для менеджеров»

Готовая страница лендинга публикуется **через Git** на **GitHub Pages** (без Netlify). Каждый `git push` в `main` обновляет сайт.

- **Публикация на GitHub:** [DEPLOY.md](DEPLOY.md) и [GITHUB_PAGES.md](GITHUB_PAGES.md) — включить Pages, указать папку `cursor-intensive-landing`.
- **Заявки в Notion (по желанию):** развернуть [api/send-to-notion.js](api/send-to-notion.js) на Vercel, подставить URL в лендинг — см. [GITHUB_PAGES.md](GITHUB_PAGES.md#заявки-в-notion-по-желанию).
- **Таблица Notion:** [NOTION_SETUP.md](NOTION_SETUP.md) — колонки Name, Email, Phone, Сумма, Status, Date.

Папка [cursor-intensive-landing](cursor-intensive-landing/) — HTML, стили, фото, логотипы.
