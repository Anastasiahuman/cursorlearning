# Бриф: Сторис в приложении LoyalBox

## Контекст
Мы хотим добавить формат «Сторис» в приложение — короткие полноэкранные карточки с акциями, рецептами и советами. Сейчас акции показываются баннером на главном экране, но CTR упал до 3.2% (было 5.8% в октябре). Гипотеза — формат приелся, нужен новый способ доставки контента.

## Цели
- Поднять CTR промо-контента с 3.2% до 6%+
- Увеличить время сессии на 20%
- Новый инструмент для коммерческой команды (продажа размещений поставщикам)

## Пользовательские сценарии
1. Пользователь открывает приложение → видит кружки сторис сверху главного экрана → тапает → листает
2. Пользователь смотрит сторис с акцией → свайпает вверх → переходит к товару/акции
3. Коммерческий менеджер создаёт сторис в CMS → задаёт сегмент и расписание → публикует

## Примеры контента сторис
- «Двойные баллы на фрукты до воскресенья» (акция)
- «5 рецептов из сезонных продуктов» (контент)
- «Ваш уровень: до Золотого осталось 2 300 баллов» (персонализация)
- «Danone: скидка 20% на весь ассортимент» (коммерческая интеграция)

## Ограничения
- Бюджет разработки: 3 недели (1 iOS + 1 Android + 0.5 бэкенд)
- CMS для создания сторис — отдельная задача, в первой версии через Firebase Remote Config
- Видео не поддерживаем в v1, только картинки + текст + кнопка
- Не больше 7 сторис одновременно (чтобы не перегружать)

## Метрики успеха
- CTR сторис > 6%
- Swipe-up rate > 4%
- Время сессии +20%
- Охват: 60%+ MAU видят хотя бы 1 сторис в неделю

## Референсы
- Сторис в Тинькофф (финансовые продукты)
- Сторис во ВкусВилл (рецепты + акции)
- Сторис в Яндекс Go (промо)
